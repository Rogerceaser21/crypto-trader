DT23 “Volume-Spike Confirmation” — Pine v6 (indicator + strategy)

The validated DT23 Volume-Spike Confirmation — research archetype v5F A3 — rebuilt as a causal Pine v6 indicator plus a Strategy Tester twin with an identical signal core. BTC 4h long-only volume-confirmed trend continuation: buy a strong-closing green bar on a genuine volume spike above both the 20- and 200-EMA; ride to a fixed 3R target, a fixed ATR stop, or an EMA20 close-exit. Every decision, drawing and alert fires on confirmed, closed bars only — no repaint. This page is the copy of record and the paste source for both files.

⚠ Volume feed — chart BINANCE:BTCUSDT perpetual, 4h — and nothing else.

This strategy is volume-gated, and volume is exchange-local. The entry test — volume > 2× SMA20(volume) — reads the raw feed, so a borderline spike clears the gate on one exchange and not another: the same code fires different trades on different feeds. This is not hypothetical — it happened live on the Annie fade (2026-07-20): a Bybit vector fired a trade the Binance test feed rejected. The DT23 edge was measured on BINANCE:BTCUSDT perpetual, 4h. 2 · What it does & how it trades

  • Entry signal (5 conjuncts, closed 4h bar) — volume > 2×vSMA20 (a volume spike) AND close > open (green) AND upperHalf (close in the upper half of the bar’s range) AND close > EMA20 AND close > EMA200. Long-only, flat only.
  • Fill — the open of the next 4h bar after the confirmed signal (causal, no look-ahead).
  • Fixed stop — entry − 1.5×ATR(14), with the ATR captured at the signal bar and frozen at entry (not trailing).
  • Fixed 3R target — entry + 3×(1.5×ATR) = entry + 4.5×ATR. Caps every winner — the reason the tail stays flat.
  • EMA20 close-exit — any closed 4h bar with close < EMA20 exits at market on the next open. Retires the trades that stall between stop and target.
  • Exit precedence — the intrabar stop/target (fires mid-bar) takes priority over the same bar’s close-exit (fires at the close). If one 4h bar’s range covers both stop and target, the stop is booked (worst case — a chart script cannot see the intrabar fill order; the engine’s ambiguous = loss rule).
  • Sizing of record — 1%-of-equity risk against pendRisk = 1.5×ATR (the author’s 10%-equity compounding was stripped for the honest signal test).

On-chart marks & the status panel

  • Fast / slow EMAs (aqua EMA20 / orange EMA200) — the two trend gates the entry keys off.
  • Entry triangle (green up) prints at the signal bar — the actionable moment; the fill is the next 4h open. A small label at the fill bar quotes entry / stop / target.
  • Stop line (red) and target line (green) run for the life of each trade at the fixed levels.
  • Exit × — red at a stop touch, green at a target touch, blue at the next-open fill of an EMA20 close-exit.
  • Status panel (top-right) — last-confirmed machine state: position, entry, stop, target (3R), current vol ratio (spike / normal), EMA gate state, bars in trade. A read-out, not a bar-anchored drawing.

  1. Open a BINANCE:BTCUSDT perpetual chart on the 4-hour timeframe — the exact feed and timeframe the edge was measured on. Volume is exchange-local — do not substitute another exchange.
  2. Load full history (scroll back to ~2019 or set a wide range) so the EMA200 and the 20-bar volume SMA have room to seed — the first ~200 4h bars are a warm-up and do not trade.

// ----------------------------- Per-bar indicators ----------------------------
// Computed EVERY bar in global scope so ta.* history stays intact; only READ
// inside the confirmed block, so all DECISIONS use closed-bar values.

// ----------------------------- Persistent state ------------------------------
// Pure running state (no arrays, no look-back), exactly like the Python mirror
// pine_mirror_dt23.run_state_machine — forward only. IDENTICAL to the indicator.
var int   state    = 0            // 0 flat / 1 long (LONG-ONLY)
var float entryRef = na           // entry fill (== open of the fill bar)
var float stopP    = na           // FIXED stop = entryRef - pendRisk
var float tpP      = na           // FIXED target = entryRef + reward*pendRisk
var float entryAtr = na
var float pendRisk = na
var int   sigBI    = na
var int   entryBI  = na
var int   barsIn   = 0
var bool  pendClose = false

var bool  pendEntry = false
var float pendAtr   = na
var int   pendSig   = na

// per-bar event flags (reset each bar; drive marks + alerts + orders)
signalLong    = false
enterLongFill = false
exitStopEv    = false
exitTgtEv     = false
exitEmaEv     = false
float exitPx  = na

// process CONFIRMED bars only -> no repaint, deterministic
if barstate.isconfirmed
    // (1) pending EMA20 close-exit armed at b-1 -> market fill at THIS open ------
    if state == 1 and pendClose
        exitEmaEv := true
        exitPx := open
        state := 0
        pendClose := false

    // (2) resolve a pending ENTRY armed at b-1 -> fill at THIS open (if flat) ----
    if pendEntry and state == 0
        entryRef := open
        entryAtr := pendAtr
        pendRisk := i_atrStop * entryAtr
        stopP    := entryRef - pendRisk
        tpP      := entryRef + i_reward * pendRisk
        sigBI    := pendSig
        entryBI  := bar_index
        barsIn   := 0
        state    := pendRisk > 0 ? 1 : 0
        if state == 1
            enterLongFill := true
    pendEntry := false

    // (3) intrabar FIXED stop/target on THIS bar (incl. the fill bar) -----------
    if state == 1
        stopHit = low <= stopP
        tpHit   = high >= tpP
        if stopHit and tpHit
            exitStopEv := true                   // both-in-range -> worst case = STOP
            exitPx := stopP
            state := 0
        else if stopHit
            exitStopEv := true
            exitPx := stopP
            state := 0
        else if tpHit
            exitTgtEv := true
            exitPx := tpP
            state := 0

    // (4) still open at the CLOSE & close < ema20 -> arm EMA-exit for b+1 -------
    if state == 1 and close < ema20v
        pendClose := true

    if state != 0
        barsIn := barsIn + 1

    // (5) flat at the CLOSE -> longCond arms the entry for b+1 ------------------
    if state == 0 and longCond and bar_index >= i_warmup and (not na(atrv)) and i_atrStop * atrv > 0
        signalLong := true
        pendEntry  := true
        pendAtr    := atrv
        pendSig    := bar_index

             style = label.style_label_up, color = color.new(color.green, 0), textcolor = color.white, size = size.tiny)

    // ---- STRATEGY ORDERS (submitted at the fill bar -> Tester fills next open;
    //      the documented 1-bar Tester lag). FIXED stop/target bracket refreshed

    if enterLongFill and not na(pendRisk)

    // FIXED stop + 3R target bracket while a position is open
    if state == 1 and not na(stopP) and not na(tpP)
        strategy.exit("bracket", "DT23", stop = stopP, limit = tpP, comment = "ATR stop / 3R target")
    // EMA20 close-exit -> flatten at the next open
    if exitEmaEv
        strategy.close("DT23", comment = "EMA20 close-exit")
