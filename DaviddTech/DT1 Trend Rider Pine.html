DT1 “Trend Rider” — Pine v6 (indicator + strategy)

The validated DT1 Trend Rider — DaviddTech’s library strategy C23-4h tunable — rebuilt as a causal Pine v6 indicator plus a Strategy Tester twin with an identical signal core. BTC 4h EMA-cross (20/75) with a long-term macro trend gate (EMA 1200) and a fixed 3×ATR(6) stop. Every decision, drawing and alert fires on confirmed, closed bars only — no repaint. This page is the copy of record and the paste source for both files.

Tested feed — chart this exact one: BINANCE:BTCUSDT perpetual, 4h. The edge was measured on Binance BTCUSDT-perp candles. EMA-cross timing is price-local: a different exchange’s prices cross at slightly different bars and will not reproduce the record. Chart the same feed the backtest used. 2 · What it does & how it trades

  • Entry signal — ta.crossover(EMA20, EMA75) for longs, ta.crossunder for shorts, on a closed 4h bar.
  • Macro regime gate — longs only when close is above the EMA 1200; shorts only when below. Evaluated at the signal bar.
  • Fill — the open of the next 4h bar after the confirmed signal (causal, no look-ahead).
  • Opposite cross always closes the open position at that next open — even when the macro gate blocks a new entry in the opposite direction.
  • Hard stop — 3.0×ATR(6) fixed from the entry price. Detected on the chart 4h high/low (identical exit bar to the engine’s 1m detection — a 4h low is the min of its 1m lows).
  • No take-profit — trades exit only by the opposite cross or the ATR stop. Trends are ridden to the reversal.
  • Sizing of record — 1%-of-equity risk against the stop distance (the strategy twin sizes the same way; the library’s 0.3%/25%-notional caps were standardised away for the signal test).

On-chart marks & the status panel

  • Fast / slow / macro EMAs (aqua / orange / gray) — the three moving averages the system keys off.
  • Entry triangle (green up / red down) prints at the signal bar — the actionable moment; the fill is the next 4h open. A small label at the fill bar quotes the entry price and stop.
  • Stop line (red) runs for the life of each trade at the fixed 3×ATR stop.
  • Exit × — red at the bar an ATR stop is hit, blue at the next-open fill of an opposite-cross exit.
  • Status panel (top-right) — last-confirmed machine state: position, side, entry, stop, regime (bull/bear), bars in trade. A read-out, not a bar-anchored drawing.

The 1-bar Strategy-Tester lag caveat — read before judging the Tester’s P&L. The number of record is our Python backtest (+0.896R center), which fills at the open of the bar after the confirmed crossover and detects stops on 1m granularity. The Strategy-Tester twin submits its market order the moment that entry bar confirms, so TradingView fills it on the following open — about one 4h bar later than the engine — and resolves stops on 4h bars. Stop levels use the engine geometry (fill ∓ 3×ATR), sizing is 1% risk. TradingView also caps 4h history, so you see recent years, not the full 2019–2024 record. The chart is for SEEING the trades; the authoritative numbers are the backtest reports on this site.

4 · Install on TradingView

  1. Open a BINANCE:BTCUSDT perpetual chart on the 4-hour timeframe — the exact feed and timeframe the edge was measured on.
  2. Load full history (scroll back to ~2019 or set a wide range) so the macro EMA 1200 has room to seed — the first ~1200 4h bars are a warm-up and do not trade.
  3. Open Pine Editor, paste the indicator source (copy button below), Save, then Add to chart. It overlays the EMAs, marks entries/exits and shows the status panel.
  4. For the Strategy Tester picture, add the strategy source as a second script — identical signals, plus the trade list and P&L (mind the 1-bar Tester-lag caveat above).
  5. To alert: right-click the chart → Add alert → Condition = this indicator → pick “LONG signal”, “SHORT signal”, “entry (either side)” or “exit” → Once per bar close. Entry alerts fire on the crossover bar; enter at the next 4h open.
  6. Leave the signal knobs on their validated center defaults. They are exposed as inputs so you can inspect — not so you re-tune the edge.

// process CONFIRMED bars only -> no repaint, deterministic
if barstate.isconfirmed
    // (1) resolve the pending signal (from bar b-1) at THIS bar's OPEN ----------
    if pendDir != 0
        // opposite cross ALWAYS closes the open position (unconditional on gate)
        if state != 0 and state == -pendDir
            exitCrossEv := true
            exitPx := open
            state := 0
        // new entry only if flat now AND the macro gate was open at the signal bar
        if state == 0 and pendGate
            entryRef := open
            entryAtr := pendAtr
            stopP    := pendDir == 1 ? entryRef - i_atrMult * entryAtr : entryRef + i_atrMult * entryAtr
            entryBI  := bar_index

        exitPx := stopP
        state := 0
    else if state == -1 and high >= stopP

        exitPx := stopP
        state := 0

var float entryRef = na           // entry fill (== open of the fill bar)
var float stopP    = na           // fixed hard stop (entryRef -/+ atrMult*ATR)
var float entryAtr = na
var int   entryBI  = na

// process CONFIRMED bars only -> no repaint, deterministic
if barstate.isconfirmed

    if pendDir != 0
        // opposite cross ALWAYS closes the open position (unconditional on gate)
        if state != 0 and state == -pendDir
            exitCrossEv := true
            exitPx := open

        // new entry only if flat now AND the macro gate was open at the signal bar
        if state == 0 and pendGate
            entryRef := open
            entryAtr := pendAtr
            stopP    := pendDir == 1 ? entryRef - i_atrMult * entryAtr : entryRef + i_atrMult * entryAtr
            entryBI  := bar_index
            barsIn   := 0
            state    := pendDir
            if pendDir == 1
                enterLongFill := true
            else
                enterShortFill := true
        pendDir  := 0
        pendGate := false
        pendAtr  := na

    // (2) stop check for the open position on THIS bar (incl. entry/signal bar) --
    if state == 1 and low <= stopP
        exitStopEv := true
        exitPx := stopP
        state := 0
    else if state == -1 and high >= stopP
        exitStopEv := true
        exitPx := stopP
        state := 0

    canSig = bar_index >= i_warmup and not na(atrv)

        crossExitSig := true
    if canSig and longSig
        signalLong := regBull
        pendDir  := 1
        pendGate := regBull
        pendAtr  := atrv
    else if canSig and shortSig
        signalShort := regBear
        pendDir  := -1

        pendAtr  := atrv

    if enterLongFill and not na(stopP)

    else if enterShortFill and not na(stopP)

        q  = sd > 0 ? strategy.equity * i_riskPct / 100.0 / sd : 0.0

        strategy.close("DT1", comment = "opposite cross")
