Annie's Monday Fade 2 — Full System Backtest

The complete weekly system, exactly as Igor ruled it (13 rulings, 2026-07-14, incl. the Ruling-13 revision: entry trigger = V1's 50-EMA vector cross ONLY). Every ruled component is wired into one pre-registered 2×2×4 = 16-cell grid and run in-sample only. The validated Fade-1 entry chain is the spine (the 91 trades); the grid tests three "upgrade" layers on top — a spike qualifier (Weekend-SD cross), a 4th retest confluence (keyless liquidity proxy), and a level-based exit (SVC + midweek reversal + Friday) against the two fixed targets. BTC-USDT perp, 15m timing / 1H structure. In-sample 2019-10-20 → 2024-12-31; the locked 2025-26 out-of-sample block is sliced off and never touched.

Verdict

The full Fade 2 does not beat the simple validated fade. All three upgrade layers fail to add value in-sample.

The elaborate machinery Igor described — count three levels, wait for the SVC, exit on the midweek reversal, gate on the Weekend-SD spike, add a liquidity confluence — is all faithfully coded here and it underperforms a plain fixed target on the same 91 entries. Concretely:

  • The level exit LOSES to the fixed target. Best level cell = +0.334R/trade (LEVEL + 50%-off-at-1.9R then breakeven); the plain 2.5R target = +0.506R, the 1.9R Fade-1 anchor = +0.382R. The clever exit gives up ~0.17R per trade. What it buys is lower drawdown and a higher win rate (3.7R DD / 59% WR vs 6.1R / 54%), not more money. Fade 2's value, if any, is variance reduction — not expectancy.
  • The Weekend-SD spike qualifier is a filter trap. It cuts the book from 91 → 73 trades (−20%) while per-trade expectancy stays flat (+0.382 → +0.369R at 1.9R). Total R falls with it. Exactly the F2 lesson — fewer trades, no better edge. Drop it.
  • The 2-of-4 liquidity confluence is inert. It changed zero entries (ledger byte-identical to 2-of-3). Structural reason below: the 50-EMA retest bar already carries confluence ②, so a 4th vote is never needed.

The best configuration on this data remains the one already known: the validated fade with a fixed 2.5R target (the F3 candidate, +0.506R, +46.0R total, t=3.35). If Igor prefers a smoother equity curve over peak return, the level exit is a defensible, remarkably robust low-variance alternative (P(ruin)=0, stable under every sensitivity perturbation) — but he should choose it with eyes open that it earns less.

3 · What each upgrade layer actually did

a) The level exit under-earns the fixed target — honestly

This was the flagship hypothesis: hold through the three levels and exit on the real midweek reversal instead of a mechanical R multiple. Coded exactly to the Q8 ruling — exit on the first of (a) an SVC after ≥2 confirmed F6 1H-levels at a weekly extreme or an active liq-proxy level, (b) a Wed/Thu-NY opposite M/W/FD via the F7 detector, or (c) the Friday cap, with the hard stop live throughout. It fires — 58-60 of 91 trades exit on the level rule, not on the stop or Friday — so the machinery is genuinely active, not a pass-through. But it exits too early on the winners: the midweek reversal and the SVC both trigger before a clean 2.5R has been paid, so mean R drops to +0.325/+0.334 versus +0.506 for the plain 2.5R. This is the F7 result, confirmed inside the full system: the fixed target beats the level exit. The consolation is real but modest — the level exit lifts win rate to 56-59% and cuts max drawdown to 3.7R (from 6.1R at 2.5R), so it is the lower-variance choice, just not the higher-return one.

b) The Weekend-SD spike qualifier is a filter trap

Gating on "the Monday spike must cross the smallest of the three Weekend-SD average lines" (History const, live 4-Monday mean, live 13-Monday mean, all measured causally from the Sun-22:00-UTC dead-gap base) removes 18 of 91 weeks — the small-Monday-move weeks whose excursion never reaches ~2.8%. Per-trade expectancy barely moves (+0.382 → +0.369R at 1.9R; a hair better at 2.5R, +0.506 → +0.532R), so the filter buys no edge — it just shrinks the book and the total R with it. It does tighten PF and drawdown at 2.5R (2.34 → 2.58; 6.1R → 4.2R), a mild quality tilt, but not one worth forfeiting a fifth of the trades. This is the F2 lesson verbatim: cutting N without lifting expectancy is a trap, not a filter.

c) The 2-of-4 liquidity confluence is inert (and here's exactly why)

Wiring the keyless liq-proxy (previous wicks + unrecovered vectors, task F12) in as the ruled 4th retest confluence changed nothing — the 2-of-4 ledger is byte-identical to 2-of-3. The reason is structural, not a bug: the retest menu's own gate is the 50-EMA touch, and L1 is by definition the vector that broke across that 50-EMA, so a bar that wicks back to the EMA almost always re-enters the L1 vector's range too. Confluence ② (return into the L1 range) is present at all 91 retests. With ② already satisfied at the first retest bar, the menu reaches its "2 of N" the instant it touches the EMA — a 4th vote can never be the tie-breaker, so the liq test is never even evaluated. The proxy is fine; it simply has no room to act inside this particular menu.

6 · Honest limits

Read this before acting on any number above.
  • In-sample only. Everything here is 2019–2024. The 2025-26 block is locked and untouched — no conclusion here is out-of-sample validated. The eventual single coordinated OOS event decides whether any of it holds.
  • 91 trades is thin for a weekly system, and the SD-cross slice (73) is thinner. Paired-t of the level exit vs 2.5R (−1.6 to −2.1) is directionally clear but not past the 2.5 bar; treat "level < fixed" as a strong lean, not a proof.
  • The level/SVC/liq detectors are proxies, not Annie's eye. The Fade-2 spec itself rates the level count at 45–53% agreement with Igor's labels and the SVC at 61%. The level exit inherits that fuzz — a human reading "this is L3, this is the reversal" may well beat the mechanical version. The backtest measures the coded system, not the discretionary one.
  • The liq proxy is untested as a level source here — it never got to act (§3c), so this run says nothing about whether the proxy is any good, only that the 2-of-4 menu had no room for it.
  • Direction is still the campaign's weak point (coded fade direction ≈ coin-flip vs Igor). Fade 2 inherits Fade 1's entries wholesale, so it inherits that ceiling; none of the exit or confluence work touches it.

7 · Method, invariants & reproducibility

  • Nothing validated was modified. New file backtests/bt_fade2.py imports the entry detector (bt_weekly2, l1_mode a = Ruling 13), the fixed/partial exit machinery (bt_weekly2_exits, F3), the midweek reversal detector (bt_weekly2_midweek_exit, F7), the 1H level detector (detectors/levels_fade2, F6 winning cell 2·2·0.02), the calibrated SVC (detectors/svc, engulfing counts <1H), and the keyless liq proxy (detectors/liq_proxy, F12).
  • Fill conventions are bt_weekly verbatim: entry at the next 15m open, stop/target filled at level with adverse slippage, 1m intrabar resolution, same-minute stop&target ambiguity counted a loss, fees 0.05%/side, slippage 0.02%/fill, 1R = 1% of $10k fixed (no compounding). The hard stop is live under every exit variant.
  • Everything causal. The SD live means use only prior completed Mondays; the liq proxy reads only candles strictly before the query bar; the level detector emits a level only once its consolidation is confirmed; the midweek reversal is confirmed k bars after its pivot. No future data enters any decision.
  • Declared grid only. A {none, SD-cross} × B {2-of-3, 2-of-4} × C {1.9R, 2.5R, LEVEL, LEVEL+BE}; all thresholds (LIQ 0.35%, weekly-extreme 0.5%, ≥2 levels, F6 2·2·0.02, SVC 150%/wick>body/2×vol) pre-registered in the file header, not fit to the result.
