2 · What it does & how it trades

  • Detrended oscillator — osc = nz(close − SMA200), RMS-normalised: norm = osc / oscRms. This strips the trend and leaves the cycle.
  • Phase-locked loop — a recursive PLL locks its phase and frequency onto the ~128-bar cycle; sinP = sin(phase) is the reconstructed cycle wave, and lockQ measures how cleanly it is locked.
  • Entry signal — ta.crossover(sinP, 0) for longs, ta.crossunder(sinP, 0) for shorts, on a closed 1h bar.
  • Lock gate — an entry opens only when lockQ > 1.4 and the PLL is warmed up (bar > 300). Weak-lock crosses are ignored for entries.
  • Opposite zero-cross always closes the open position at the next open — even when unlocked. A close + open on the same bar is a reversal.
  • Fill — the open of the next 1h bar after the confirmed signal (causal, no look-ahead).
  • Hard bracket — long: stop = fill × (1−4%), target = fill × (1+12%); short mirrored. Fires on any bar from entry onward, detected on the chart 1h high/low. The hard stop is what caps the ETH short tail that destroyed the stop-less DT10/DT11.
  • Ambiguous bar = STOP — if one 1h bar spans both the stop and the target, this chart-TF script books the stop (worst case) — TradingView’s OCO default and the engine’s “ambiguous = loss” rule. On the cp=128 ledger no exit bar spanned both, so the mirror still reproduces it 151/151.
  • Sizing of record — flat 1%-of-equity risk against the 4% stop distance (the strategy twin sizes the same way).

On-chart marks & the status panel

  • Entry triangle (green up / red down) prints at the signal bar — the actionable ready&locked zero-cross; the fill is the next 1h open. A small label at the fill bar quotes entry, stop and target.
  • Stop line (red) and target line (lime) run for the life of each trade at the fixed ±% bracket.
  • Exit × — red at a stop hit, lime at a target hit, blue at the next-open fill of an opposite zero-cross exit.
  • Optional pane — sinP (the cycle wave) and lockQ in the data window, purely informational (never a decision input).
  • Status panel (top-right) — last-confirmed machine state: position, side, entry, stop, target, lockQ (locked/unlocked), and the cycle sinP (up-half / down-half). A read-out, not a bar-anchored drawing.

Why the match is exact despite the chart-TF bracket: every market fill is the next 1h open (a 1h open is its first 1m open), and a bracket’s exit bar is timeframe-invariant (a 1h high is the max of its 1m highs). Only a bar that spans both levels could differ — the engine used 1m to tie-break, the Pine takes the stop — and on the cp=128 ledger zero exit bars did that. So the Pine is byte-for-byte here and strictly more conservative in general.

The 1-bar Strategy-Tester lag caveat — read before judging the Tester’s P&L. The number of record is our Python backtest (+0.281R at cp=128), which fills at the open of the bar after the confirmed zero-cross. The Strategy-Tester twin submits its market order the moment that entry bar confirms, so TradingView fills it on the following open — about one 1h bar later than the engine. Bracket levels use the engine geometry (fill ±4% / +12%), sizing is 1% risk, and when a bar spans both levels TradingView assumes the stop (matching the engine). TradingView also needs full history for the PLL recursion and caps 1h history, so you may see fewer years than the full 2019–2024 record. The chart is for SEEING the trades; the authoritative numbers are the backtest reports on this site.

// ----------------------------- Detrended RMS oscillator ----------------------
// Computed EVERY bar in global scope so ta.* history stays intact; only READ
// inside the confirmed block, so all DECISIONS use closed-bar values.
trend  = ta.sma(close, i_detrend)
osc    = nz(close - trend)                               // nz -> 0 during SMA warmup
oscsq  = osc * osc
oscRms = math.sqrt(math.max(nz(ta.sma(oscsq, i_detrend)), 1e-10))   // 1e-10 floor on the MEAN
norm   = osc / oscRms

twoPi   = 2.0 * math.pi
nomFreq = twoPi / i_centerPeriod
fMax    = nomFreq * 2.0
fMin    = nomFreq * 0.4
Kp      = 1.4142 * i_loopBW
Ki      = i_loopBW * i_loopBW

// ----------------------------- Persistent PLL + trade state ------------------
// Pure running state (no arrays, no look-back) — the exact recursion the engine
// evolves bar-by-bar from bar 0. Advanced ONLY under barstate.isconfirmed.
var float phase      = 0.0
var float integrator = 0.0
var float iLpf       = 0.0
var float qLpf       = 0.0
var float sinPrev    = 0.0            // sinP of the previous confirmed bar (for the cross)

var int   state    = 0               // 0 flat / +1 long / -1 short
var float entryRef = na              // entry fill (== open of the fill bar)
var float stopP    = na
var float targP    = na
var int   entryBI  = na
var int   barsIn   = 0
var float lockAtSig= na              // lockQ at the entry signal bar

var int   pendDir   = 0              // a sinP zero-cross fired on the PREVIOUS confirmed bar
var bool  pendReady = false          // was bar ready (bar_index>300) at that signal bar?
var bool  pendLock  = false          // was lockQ > lockThresh at that signal bar?

// live read-outs for the status panel (updated each confirmed bar)
var float lockQv = 0.0
var float sinPv  = 0.0

// per-bar event flags (reset each bar; drive marks + alerts)
signalLong   = false                 // gated LONG entry signal, AT the signal bar (actionable)
signalShort  = false                 // gated SHORT entry signal, AT the signal bar (actionable)
crossExitSig = false                 // opposite zero-cross while in a trade, AT the signal bar
enterLongFill  = false               // LONG fill recorded, AT the fill bar
enterShortFill = false               // SHORT fill recorded, AT the fill bar
exitBrkStop  = false                 // stop bracket hit, AT the breach bar
exitBrkTp    = false                 // target bracket hit, AT the breach bar
exitCrossEv  = false                 // zero-cross close, AT the next-open fill bar
float exitPx = na

// process CONFIRMED bars only -> no repaint, deterministic
if barstate.isconfirmed

    // opposite zero-cross ALWAYS closes; a ready&locked signal opens; both = reversal.
    if pendDir != 0
        if state != 0 and state == -pendDir
            exitCrossEv := true
            exitPx := open
            state := 0
        if pendReady and pendLock and ((pendDir == 1 and state <= 0) or (pendDir == -1 and state >= 0))
            entryRef := open
            stopP := pendDir == 1 ? entryRef * (1.0 - i_slPct / 100.0) : entryRef * (1.0 + i_slPct / 100.0)
            targP := pendDir == 1 ? entryRef * (1.0 + i_tpPct / 100.0) : entryRef * (1.0 - i_tpPct / 100.0)
            entryBI := bar_index
            barsIn  := 0
            state   := pendDir
            if pendDir == 1
                enterLongFill := true
            else
                enterShortFill := true
        pendDir   := 0
        pendReady := false
        pendLock  := false

    // (2) intrabar bracket check on THIS 1h bar (incl. entry bar). Both-touch ->
    //     STOP (worst case; ambiguous = loss). Bracket takes priority over the
    //     close-based zero-cross on the same bar.
    if state != 0 and not na(stopP)
        stopHit = state == 1 ? low <= stopP : high >= stopP
        tpHit   = state == 1 ? high >= targP : low <= targP
        if stopHit
            exitBrkStop := true
            exitPx := stopP
            state := 0
        else if tpHit
            exitBrkTp := true
            exitPx := targP
            state := 0

    if state != 0
        barsIn := barsIn + 1

    // (3) advance the PLL recursion at the CLOSE of this bar --------------------
    warmed = bar_index > i_warm
    cosRef = math.cos(phase)
    sinRef = math.sin(phase)
    iInst = warmed ? norm * cosRef : 0.0
    qInst = warmed ? -norm * sinRef : 0.0
    iLpf := iLpf + i_lpfAlpha * (iInst - iLpf)
    qLpf := qLpf + i_lpfAlpha * (qInst - qLpf)
    denom = math.sqrt(iLpf * iLpf + qLpf * qLpf) + iLpf
    phaseErr = denom > 1e-10 ? 2.0 * math.atan(qLpf / denom) : (qLpf >= 0 ? math.pi : -math.pi)
    integrator := warmed ? integrator + Ki * phaseErr : 0.0
    freqRaw = nomFreq + Kp * phaseErr + integrator
    freqClamped = math.min(fMax, math.max(fMin, freqRaw))
    if freqClamped != freqRaw
        integrator := freqClamped - nomFreq - Kp * phaseErr
    phase := phase + freqClamped
    if phase >= twoPi
        phase := phase - twoPi
    if phase < 0.0
        phase := phase + twoPi
    lockQ = math.sqrt(iLpf * iLpf + qLpf * qLpf) / 0.7071
    sinP  = math.sin(phase)
    lockQv := lockQ
    sinPv  := sinP

        pendDir   := 1
        pendReady := rdy
        pendLock  := lk
    else if shortSig
        signalShort := rdy and lk
        pendDir   := -1
        pendReady := rdy
        pendLock  := lk
