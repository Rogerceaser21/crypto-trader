Annie Monday Fade — V3 STREAMING

The validated Annie Monday weekend-box fade rebuilt as a streaming Pine v6 indicator: it commits to the running early-week extreme in real time and fires the alert at the entry bar — the signal a live trader gets is the signal that was backtested. Causal, no repaint. Live-honest in-sample edge: +0.347R/trade × 167 Monday trades at the 2.5R target (t=2.84, ~2.7 trades/mo). This page is the copy of record and the paste source.

1 · Why V3 is streaming (and V1 wasn’t tradeable live)

V1 could only tell you which early-week spike was the spike once the early window closed (~Tuesday 17:00 NY). So its alert fired late — after the trade was often already gone — and when we measured that lock-at-close timing honestly it made almost nothing (+0.04R, t=0.36). V3 runs the setup the way you actually trade it: it commits to the running early-week extreme the moment the spike gate passes, and it fires at the entry bar. Every time a bigger extreme appears, the trigger chain resets to it, in real time. That streaming form is the one that keeps the edge, and it is what this indicator draws and alerts on. Nothing is drawn in hindsight: a printed signal is fixed the instant it prints and never moves.

4 · What it draws

The indicator is an overlay. When a streaming fade resolves it marks the whole setup at its true historical bars and alerts on the entry bar. Built for 15m BTCUSDT perp.

  • 50-EMA (orange) — the 15m 50-period EMA the setup keys off.
  • Weekend box (grey) — the Fri 5pm→Sun 5pm NY close-range the week spikes out of.
  • Spike peak (▲/▼ peak) — the committed early-week extreme being faded.
  • L1 vector (yellow box) — the first 200%-class PVSRA vector whose body closes across the 50-EMA in the fade direction.
  • Retest label — the 2-of-3 retest bar that arms the entry.
  • Entry arrow (green LONG / red SHORT) — at the next 15m open, the fill of record.
  • Weekly-extreme line (grey dotted) — the running extreme the stop sits beyond.
  • Stop & TP lines (dashed red / green) — stop the extreme ±0.05%, target the selected R multiple.

Every visual has its own toggle in Settings → Visuals. The green/red triangle on the entry bar is the bar the alertcondition fires from.

New in V3.2 — fully-causal display, both files (signals unchanged)

  • Nothing is ever drawn at a past bar. Every mark prints at the confirmed bar the machine learned it: grey ▲/▼ peak? at the commit bar (that bar is the extreme), yellow L1 box at the L1 bar the moment it qualifies, blue retest tag at the retest bar, entry label at the entry bar (the faded peak’s price is quoted in its text). Stop/TP/extreme lines project forward only. The old entry-time habit of marking the resolved peak/L1/retest back at their historical bars is removed. Printed marks never move and are never deleted.
  • Trail throttle — on trending Mondays the running extreme extends dozens of times; the machine still re-commits on every one (logic identical, streaming), but a new grey label prints only on the week’s first commit, a side flip, or an extension ≥0.25% beyond the last label (input, adjustable). Kills the label cascade.
  • Status panel (top-right) — the machine’s last-confirmed state: phase (waiting spike → peak ✓ waiting L1 → L1 ✓ hunting retest → ARMED → ENTERED / chain dead), fade side, committed peak price, Monday-gate check, early-window clock, weekend-box bounds. A read-out, not a drawing — it cannot repaint history.
  • Alert “peak committed (streaming)” fires on labeled commits. The entry alert remains the only trade signal.
  • The Strategy-Tester twin carries the identical V3.2 visual layer — trail, causal marks, panel, alert.

5 · Install on TradingView

  1. Open a BTCUSDT perpetual chart on the 15-minute timeframe (Binance/BitMEX/Bybit perp; the edge is measured on perp).
  2. Open Pine Editor (bottom of the TradingView screen).
  3. Click Copy Pine source below, paste it over anything in the editor, and press Save (name it whatever you like).
  4. Click Add to chart. The indicator appears as an overlay on price and the setup draws as each week resolves.
  5. To alert: right-click the chart → Add alert → Condition = this indicator → pick “V3 — LONG entry”, “SHORT entry”, or “entry (either side)” → Once per bar close. It fires on the entry bar, live.
  6. Tune the TP (R multiple) input if you want — 1.9 / 2.5 (default) / 3.0. Leave the detector knobs on their validated defaults.

Find it later under Indicators → My scripts (the indicators you saved).

//@version=6

// Annie Monday Fade — V3  ·  STREAMING   (live-executable rebuild)
// -----------------------------------------------------------------------------
// The validated Annie "Weekly Setup / FMWB fade" on 15m BTCUSDT-perp, rebuilt so
// the signal a live trader gets is the signal that was actually backtested.
//
// WHY V3 EXISTS (the V1 problem, in one line): the V1 indicator/strategy only
// knew "the" early-week spike once the early window CLOSED (~Tue 17:00 NY), so its
// alert fired late — un-actionable live, and its lock-at-close economics were dead
// (+0.04R, t=0.36). V3 runs the setup the way you actually trade it: it commits to
// the RUNNING early-week extreme in real time and fires AT the entry bar. This
// streaming form is the one that keeps the edge:
//   167 in-sample Monday trades (2019-09→2024-12) · +0.347R/trade @ 2.5R target ·
//   WR 45.5% · PF 1.64 · t=2.84 · ~2.7 trades/mo.  (@1.9R: +0.257R, t=2.46, PF 1.51.)
//   The old +0.382R figure was HINDSIGHT-mode (you-knew-the-spike) — shown as a
//   caveat only; the honest, live-executable number is +0.347R.
//
// NO REPAINT: every decision uses CONFIRMED, closed bars only (barstate.isconfirmed).
// A printed signal never moves or disappears — the retest resolves on bar rt, the
// entry fills on the NEXT 15m open (rt+1), and both are drawn at their true bars.
// No request.security, no future references.
//
// V3.2 FULLY-CAUSAL DISPLAY (2026-07-20, Igor's order — signals UNCHANGED):
// EVERY mark prints AT the confirmed bar where the machine learned it, and no
// drawing is ever positioned at a past bar:
//   · gray "peak?" trail at each commit bar (the commit bar IS the extreme bar),
//     THROTTLED: re-label only on side flip or when the committed extreme moved
//     >= i_trailThrottle beyond the last labeled one (kills trend-day label spam;
//     commits themselves are NOT throttled — logic identical);
//   · yellow L1 box at the L1 bar the moment it qualifies;
//   · blue "retest" tag at the retest bar the moment it completes;
//   · entry label at the entry bar (peak price quoted in its text instead of a
//     backwards-drawn marker), stop/TP/extreme lines projected FORWARD only.
// Plus the top-right STATUS PANEL (last-confirmed machine state) and a "peak
// committed" alert that fires on labeled commits. The V3.0/3.1 entry-time habit
// of drawing the resolved peak/L1/retest back at their historical bars is GONE —
// Igor's rule: nothing retro-positioned, ever. Old labels never move or delete.
//
// THE SETUP (per week, Sun 17:00 NY → next Sun 17:00 NY):
//   Weekend box (Fri17:00→Sun17:00 NY close range) → an early-week (Sun/Mon/Tue)
//   spike OUT of the box. STREAMING: the faded side is the side of the RUNNING
//   extreme right now; every time that running extreme changes the L1→retest chain
//   RESETS to it. FADE it (spike UP → SHORT, spike DOWN → LONG) → L1 = first
//   200%-class PVSRA vector whose body closes across the 15m 50-EMA in the fade
//   direction after the extreme → a 2-of-3 retest {50-EMA wick-touch + body holds,
//   return into the L1-vector range} on the trade side (abort if a bar closes back
//   across the EMA first) → ENTER the NEXT 15m open → stop the running weekly
//   extreme ±0.05%, TP a selectable R multiple (default 2.5R). Monday-spike only.
//
// FIDELITY: backtests/g1_pine_mirror_v3.py re-implements THIS script's exact
// forward state machine in Python and matches the 167-trade streaming ground truth
// 159/167 = 95.2% exact (entry bar + side + stop + TP identical), every one of the
// 14 residuals traced to a named platform quirk of the reference (RW: its 40-day
// warmup + dropped final week; A1: its fixed-7-day week stepping drifts ±1h across
// DST while a live chart re-anchors to Sun-17:00-NY weekly). Deterministic; the
// acid test fires this build's LONG at Tue 2026-07-14 01:15 UTC — Igor's manual
// trade — on live keyless Binance data.
//
// PVSRA 200%-class vector (== detectors/pvsra.py): volume ≥ 2.0× the average of the
// PRIOR 10 candle volumes (ta.sma(volume[1],10)); green if close≥open, red if not.
// Chart: BTCUSDT perp, 15m.
// =============================================================================

indicator("Annie Monday Fade — V3 STREAMING", overlay = true,
     max_lines_count = 500, max_labels_count = 500, max_boxes_count = 500)

// ----------------------------- Inputs ----------------------------------------
gA = "Anchors (Igor's weekly model)"
i_tz       = input.string("America/New_York", "Anchor timezone", group = gA)
i_openHour = input.int(17, "Week open hour (Sunday, NY)", minval = 0, maxval = 23, group = gA)
i_friHour  = input.int(8,  "Friday hard-cutoff hour (NY)", minval = 0, maxval = 23, group = gA)
i_earlyFrac= input.float(0.286, "Early-window fraction of week (Sun/Mon/Tue)", minval = 0.1, maxval = 0.6, step = 0.001, group = gA)

gD = "Detector knobs (validated defaults — do not change to trade the edge)"
i_emaLen   = input.int(50, "50-EMA length", minval = 2, group = gD)
i_volLb    = input.int(10, "PVSRA volume lookback (prior N)", minval = 2, group = gD)
i_vecMult  = input.float(2.0, "200%-class vector threshold (x vol avg)", minval = 1.0, step = 0.1, group = gD)
i_minSpike = input.float(0.10, "Min spike beyond box (%)", minval = 0.0, step = 0.01, group = gD)
i_touchTol = input.float(0.15, "50-EMA wick-touch tolerance (%)", minval = 0.0, step = 0.01, group = gD)
i_crossTol = input.float(0.10, "50-EMA body-cross tolerance (%)", minval = 0.0, step = 0.01, group = gD)
i_span     = input.int(320, "Max bars L1 -> retest", minval = 1, group = gD)
i_stopBuf  = input.float(0.05, "Stop buffer beyond extreme (%)", minval = 0.0, step = 0.01, group = gD)
i_rr       = input.float(2.5, "TP (R multiple)", options = [1.9, 2.5, 3.0], group = gD)
i_mondayOnly = input.bool(true, "Monday-spike only (the validated edge)", group = gD)

gV = "Visuals"
i_showBox    = input.bool(true,  "Weekend box", group = gV)
i_showL1     = input.bool(true,  "L1 vector highlight", group = gV)
i_showRetest = input.bool(true,  "Retest marker", group = gV)
i_showEntry  = input.bool(true,  "Entry arrow", group = gV)
i_showStopTP = input.bool(true,  "Stop / TP lines", group = gV)
i_showExt    = input.bool(true,  "Weekly-extreme line", group = gV)
i_showEma    = input.bool(true,  "50-EMA", group = gV)
i_showLivePeak = input.bool(true, "Live peak-commit trail (gray, causal)", group = gV)
i_trailThrottle = input.float(0.25, "Trail re-label threshold (% move of extreme)", minval = 0.0, step = 0.05, group = gV)
i_showState  = input.bool(true,  "Status panel (top-right)", group = gV)

// derived tolerances (fractions)
touchTol = i_touchTol / 100.0
crossTol = i_crossTol / 100.0
minSpike = i_minSpike / 100.0
stopBuf  = i_stopBuf  / 100.0

DAY_MS = 86400000

// ----------------------------- Per-bar indicators ----------------------------
// EMA50 == pandas close.ewm(span,adjust=False) == ta.ema(close,50)
ema50 = ta.ema(close, i_emaLen)
// 200%-class: volume >= vecMult x mean(prior i_volLb volumes) == ta.sma(volume[1],lb)
avgVol   = ta.sma(volume[1], i_volLb)
volRatio = na(avgVol) or avgVol == 0 ? 0.0 : volume / avgVol
is200    = volRatio >= i_vecMult
isUp     = close >= open

plot(i_showEma ? ema50 : na, "50 EMA", color = color.new(color.orange, 0), linewidth = 1)

// ----------------------------- Week time anchors -----------------------------
_dow  = dayofweek(time, i_tz)          // 1=Sunday .. 7=Saturday
_yr   = year(time, i_tz)
_mo   = month(time, i_tz)
_dy   = dayofmonth(time, i_tz)
_today0     = timestamp(i_tz, _yr, _mo, _dy, i_openHour, 0)
_daysBack   = _dow - 1                  // Sun=0 .. Sat=6
_approxSun  = _today0 - _daysBack * DAY_MS
weekOpenTs  = timestamp(i_tz, year(_approxSun, i_tz), month(_approxSun, i_tz), dayofmonth(_approxSun, i_tz), i_openHour, 0)
if time < weekOpenTs
    _approxSun2 = _today0 - (_daysBack + 7) * DAY_MS
    weekOpenTs := timestamp(i_tz, year(_approxSun2, i_tz), month(_approxSun2, i_tz), dayofmonth(_approxSun2, i_tz), i_openHour, 0)

weekEndTs  = weekOpenTs + 7 * DAY_MS
earlyEndTs = weekOpenTs + int(i_earlyFrac * 7.0 * DAY_MS)
_friApprox = weekOpenTs + 5 * DAY_MS
friEndTs   = timestamp(i_tz, year(_friApprox, i_tz), month(_friApprox, i_tz), dayofmonth(_friApprox, i_tz), i_friHour, 0)

newWeek = weekOpenTs != weekOpenTs[1]

// ----------------------------- Persistent streaming state --------------------
// (no per-bar arrays: the machine is pure running state, exactly like the Python
//  mirror g1_pine_mirror_v3.run_state_machine — forward only, no look-back.)
var float boxTop  = na
var float boxBot  = na
var float pendTop = na
var float pendBot = na

var float weekHigh = na          // running weekly extreme for the STOP (whole week)
var float weekLow  = na
var float earlyHigh = na         // running early-window extreme (spike/direction)
var float earlyLow  = na
var int   earlyHiT  = 0
var int   earlyLoT  = 0

var int   fadeDir   = 0          // 1 long, -1 short, 0 none
var float extPrice  = na
var int   extBar    = 0
var int   extT      = 0
var bool  committed = false
var bool  chainDead = false
var bool  entryTaken= false      // week decided (first wick-passing chain)

var bool  l1Found   = false
var int   l1Bar     = 0
var int   l1T       = 0
var float l1Low     = na
var float l1High    = na
var bool  c2latch   = false

// armed = retest completed on the PREVIOUS bar; the entry fills on THIS bar's open
var bool  armed        = false
var float armWick      = na
var int   armFadeDir   = 0
var int   armExtT      = 0
var float armExtPrice  = na
var int   armL1T       = 0
var float armL1H       = na
var float armL1L       = na
var int   armRetestT   = 0
var float armRetestEma = na

// signal outputs (for alerts)
longFired  = false
shortFired = false
peakCommitted = false            // true on the bar a LABELED peak commit prints (throttled)

// trail throttle state (display only — never read by signal logic)
var float lastLblP   = na
var int   lastLblDir = 0

// process only CONFIRMED bars -> no repaint, deterministic
if barstate.isconfirmed
    // --- New week bootstrap: freeze weekend box, reset all week state ---
    if newWeek
        boxTop := pendTop
        boxBot := pendBot
        pendTop := na
        pendBot := na
        weekHigh := na
        weekLow  := na
        earlyHigh := na
        earlyLow  := na
        earlyHiT := 0
        earlyLoT := 0
        fadeDir   := 0
        extPrice  := na
        extT      := 0
        extBar    := 0
        committed := false
        chainDead := false
        entryTaken:= false
        l1Found   := false
        l1Bar     := 0
        c2latch   := false
        armed     := false
        lastLblP  := na
        lastLblDir:= 0
        if i_showBox and not na(boxTop) and not na(boxBot)
            box.new(weekOpenTs - 2 * DAY_MS, boxTop, weekOpenTs, boxBot, xloc = xloc.bar_time,
                 border_color = color.new(color.gray, 40), bgcolor = color.new(color.gray, 90))

    // --- running weekly extreme (for the stop), updated every bar ---
    weekHigh := na(weekHigh) ? high : math.max(weekHigh, high)
    weekLow  := na(weekLow)  ? low  : math.min(weekLow, low)

    // --- weekend box accumulation for the FOLLOWING week ---
    if time >= weekOpenTs + 5 * DAY_MS and time < weekEndTs
        pendTop := na(pendTop) ? close : math.max(pendTop, close)
        pendBot := na(pendBot) ? close : math.min(pendBot, close)

    // ======================================================================
    // (1) ARMED-ENTRY resolution: a retest completed on the PREVIOUS bar, so THIS
    //     bar's open is the fill. Uses the snapshot taken at the retest bar, BEFORE
    //     this bar's own early-window update (mirrors the Python resolve order).
    // ======================================================================
    if armed and not entryTaken
        entryRef = open
        if time < friEndTs
            wickOk = armFadeDir == -1 ? armWick > entryRef : armWick < entryRef
            if wickOk
                entryTaken := true                     // valid candidate -> week decided
                // Monday-spike gate on the committed extreme's 5pm-NY trading day
                extHr = hour(armExtT, i_tz)
                extDw = dayofweek(armExtT, i_tz)       // 1=Sun..7=Sat
                isMon = (extHr < i_openHour and extDw == 2) or (extHr >= i_openHour and extDw == 1)
                if (not i_mondayOnly) or isMon
                    stopP = armFadeDir == -1 ? armWick * (1 + stopBuf) : armWick * (1 - stopBuf)
                    sd    = math.abs(entryRef - stopP)
                    if sd > 0
                        tpP    = armFadeDir == -1 ? entryRef - i_rr * sd : entryRef + i_rr * sd
                        dirCol = armFadeDir == -1 ? color.red : color.green
                        if armFadeDir == 1
                            longFired := true
                        else
                            shortFired := true
                        // ---- causal entry drawing: everything at THIS bar or
                        //      projected forward; nothing positioned at past bars ----
                        if i_showEntry
                            label.new(time, entryRef,
                                 text = (armFadeDir == -1 ? "SHORT" : "LONG") + " (peak " + str.tostring(armExtPrice, format.mintick) + ")",
                                 xloc = xloc.bar_time, style = armFadeDir == -1 ? label.style_label_down : label.style_label_up,
                                 color = color.new(dirCol, 0), textcolor = color.white, size = size.small)
                        if i_showExt
                            line.new(time, armWick, friEndTs, armWick, xloc = xloc.bar_time,
                                 color = color.new(color.gray, 20), width = 1, style = line.style_dotted)
                        if i_showStopTP
                            line.new(time, stopP, friEndTs, stopP, xloc = xloc.bar_time,
                                 color = color.new(color.red, 0), width = 1, style = line.style_dashed)
                            line.new(time, tpP, friEndTs, tpP, xloc = xloc.bar_time,
                                 color = color.new(color.green, 0), width = 1, style = line.style_dashed)
                            label.new(friEndTs, stopP, "stop", xloc = xloc.bar_time, style = label.style_label_left,
                                 color = color.new(color.red, 30), textcolor = color.white, size = size.tiny)
                            label.new(friEndTs, tpP, str.tostring(i_rr, "#.#") + "R TP", xloc = xloc.bar_time,
                                 style = label.style_label_left, color = color.new(color.green, 30),
                                 textcolor = color.white, size = size.tiny)
            else
                chainDead := true                      // no valid candidate -> state dies
        else
            chainDead := true                          // past Friday cutoff -> no candidate
        armed := false

    // ======================================================================
    // (2) EARLY-WINDOW running spike extreme + STREAMING commit
    // ======================================================================
    if not entryTaken and time <= earlyEndTs
        newExtreme = false
        if na(earlyHigh) or high > earlyHigh
            earlyHigh := high
            earlyHiT  := time
            newExtreme := true
        if na(earlyLow) or low < earlyLow
            earlyLow := low
            earlyLoT := time
            newExtreme := true
        boxOk = not na(boxTop) and not na(boxBot) and boxTop > boxBot
        if newExtreme and boxOk
            upExc   = (earlyHigh - boxTop) / boxTop
            downExc = (boxBot - earlyLow) / boxBot
            if math.max(upExc, downExc) >= minSpike
                newDir   = upExc >= downExc ? -1 : 1
                newExtT  = newDir == -1 ? earlyHiT : earlyLoT
                newExtP  = newDir == -1 ? earlyHigh : earlyLow
                if newDir != fadeDir or newExtT != extT
                    // committed extreme changed -> reset the chain to it
                    fadeDir   := newDir
                    extT      := newExtT
                    extBar    := bar_index
                    extPrice  := newExtP
                    committed := true
                    chainDead := false
                    l1Found   := false
                    c2latch   := false
                    // live peak trail (display only, THROTTLED): label the commit
                    // bar (== the extreme bar) causally, but only on the first
                    // commit of the week, a side flip, or when the extreme moved
                    // >= i_trailThrottle beyond the last LABELED one. Commits and
                    // chain resets are NOT throttled — the signal logic above ran
                    // in full either way. Labels are never moved or deleted.
                    doLbl = na(lastLblP) or fadeDir != lastLblDir or
                         math.abs(extPrice - lastLblP) / lastLblP >= i_trailThrottle / 100.0
                    if doLbl
                        peakCommitted := true
                        lastLblP   := extPrice
                        lastLblDir := fadeDir
                        if i_showLivePeak
                            label.new(time, extPrice, text = fadeDir == -1 ? "▲ peak?" : "▼ peak?",
                                 xloc = xloc.bar_time,
                                 style = fadeDir == -1 ? label.style_label_down : label.style_label_up,
                                 color = color.new(color.gray, 55), textcolor = color.white, size = size.tiny)

    // ======================================================================
    // (3) CHAIN step for THIS bar: L1 -> retest -> ARM (forward only)
    // ======================================================================
    if committed and not chainDead and not armed and not entryTaken and bar_index >= extBar
        emaOk = not na(ema50) and ema50 != 0
        if not l1Found
            brk = false
            if emaOk and is200
                if fadeDir == 1 and isUp and close >= ema50 and open <= ema50 * (1 + crossTol)
                    brk := true
                if fadeDir == -1 and not isUp and close <= ema50 and open >= ema50 * (1 - crossTol)
                    brk := true
            if brk
                l1Found := true
                l1Bar   := bar_index
                l1T     := time
                l1Low   := low
                l1High  := high
                c2latch := false
                // causal L1 mark: printed AT the L1 bar the moment it qualifies
                if i_showL1
                    box.new(time, high, time + 15 * 60000, low, xloc = xloc.bar_time,
                         border_color = color.new(color.yellow, 0), bgcolor = color.new(color.yellow, 80))
                    label.new(time, low, text = "L1", xloc = xloc.bar_time,
                         style = label.style_label_up, color = color.new(color.yellow, 40),
                         textcolor = color.black, size = size.tiny)
        else if bar_index > l1Bar
            if bar_index >= l1Bar + i_span
                chainDead := true                      // no retest within span
            else if emaOk
                if fadeDir == 1 and close < ema50 * (1 - crossTol)
                    chainDead := true                  // abort: closed back across EMA
                else if fadeDir == -1 and close > ema50 * (1 + crossTol)
                    chainDead := true
                else
                    if high >= l1Low and low <= l1High
                        c2latch := true
                    touched = fadeDir == 1 ? low <= ema50 * (1 + touchTol) : high >= ema50 * (1 - touchTol)
                    bodyOk  = fadeDir == 1 ? math.min(open, close) >= ema50 * (1 - crossTol) : math.max(open, close) <= ema50 * (1 + crossTol)
                    if touched and bodyOk and c2latch
                        // first qualifying retest -> ARM; entry fills next bar open
                        armed        := true
                        armFadeDir   := fadeDir
                        armWick      := fadeDir == -1 ? weekHigh : weekLow
                        armExtT      := extT
                        armExtPrice  := extPrice
