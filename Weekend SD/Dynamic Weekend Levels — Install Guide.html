Weekend SD Trading · LIVE indicator

Dynamic Weekend Levels — Install Guide

Install guide for “Weekend + Monday Mean + SD Levels — LIVE” — the final, agreed version of the indicator (average line per window, three toggleable sets, extras off by default). Updated 2026-07-13 with the timeframe-consistency fix: earlier pastes drew the Monday lines at different prices on 1h vs 15m. If your chart still runs an older paste, re-install from this page.

1 · What it draws

While a weekend is running you get the blue weekend high/low lines. From Monday 02:00 Dubai (the “dead gap” end) until Tuesday 00:00 Dubai you get the Monday stretch levels — how far Monday typically travels up and down from the base price. Three level sets, one line above and one below the base per set by default (6 lines):

ALL-HISTORY — corrected constants, 277 weekends 2019–2024 (offline) LAST 4 MONDAYS — computed live on the chart LAST 3 MONTHS — last 13 Mondays, computed live

Extras (+1SD, +2SD, p95 per set) exist but are OFF by default — switch them on in Settings if you want the wider fan. The live sets are causal: a Monday’s own move never feeds its own lines; stats freeze at the weekend’s base latch using only previously completed Mondays. Confluence zones from the earlier V3 draft were dropped in the final spec to keep the chart clean.

2 · The 2026-07-13 fix — why you must re-paste

The previous paste latched the Monday base (the price every level multiplies off) one bar too early: a 1h chart anchored to the price 60 minutes before the dead-gap end, a 15m chart 15 minutes before it. Whatever BTC moved in between shifted every Monday line — gray, orange and teal — by a different amount per timeframe. Verified against our 1-minute candle history:

Monday afterGray-line gap, 1h vs 15m (old code)New code
Jun 14$1,292$0

The fix latches the base at the close of the bar ending exactly at 02:00 Dubai — identical on every timeframe that aligns to the hour (1m…1h). The live 4-Monday / 3-month statistics measured their past moves from the same wrong base, so they are repaired by the same change. Bonus: on a live chart the lines no longer appear up to an hour early with a stale price.

Action needed: this fix only reaches your chart when you re-paste the source below and remove the old copy of the indicator. Nothing updates automatically.

4 · Install / re-install in TradingView

  1. Open a BTCUSDT perp chart on a 15m, 30m or 1h timeframe.
  2. If an older copy of the indicator is on the chart, remove it (hover its name → ×).
  3. Open the Pine Editor (bottom panel) → Open → New blank indicator. (To overwrite your saved old script instead, open that script here.)
  4. Select all existing code (Cmd/Ctrl + A) and delete it.
  5. Copy the full source from the block at the bottom of this page (button provided) and paste it in.
  6. Click Save, name it e.g. Weekend SD Levels LIVE, then Add to chart.
  7. Verify the fix: flip the chart between 15m and 1h. The gray / orange / teal Monday lines must sit at the same price on both. If they differ, an old copy is still on the chart.
  8. Let the chart load history: the 4-Monday set needs ~5 weekends on screen, the 3-month set ~14. Live lines stay empty until warmed up — that is normal, not a bug.

6 · Caveats

  • Timeframe: use 15m, 30m or 1h. On 1m TradingView can’t hold enough bars — the live sets never warm up. All hour-aligned timeframes now print identical levels (that is the fix).
  • 4-Monday set is jumpy by design: four observations — treat it as a fast read of “lately”, not gospel.
  • All-history stays a constant: TradingView cannot load 5+ years of 1m bars, so the gray set is pre-computed offline. If the offline study is re-run, update the input numbers.
  • No look-ahead: a Monday’s move joins the live stats only after its window fully closes (Tue 00:00 Dubai). What you see on any Monday was knowable in real time.
  • Not compiled inside TradingView: reviewed offline; if the editor flags anything on paste, send the error back for a one-round fix.

//@version=6
indicator("Weekend + Monday Mean + SD Levels — LIVE", overlay=true, format=format.price, scale=scale.right)

// ============================================================================
// LIVE rebuild of Igor's "Weekend + Monday Mean + SD Levels" (2026-07-12).
// Changes vs the original:
//  • THREE toggleable level sets per side instead of one frozen set:
//      ALL-HISTORY      — constants computed offline from every weekend since
//                         2019 (TradingView cannot load 5 years of intraday
//                         bars). Corrected Truth-Report values are defaults.
//      LAST 4 MONDAYS   — computed LIVE on the chart, self-updates weekly.
//      LAST 3 MONTHS    — last 13 Mondays, computed LIVE.
//  • Default view = the AVERAGE Monday stretch per set per side (6 lines).
//    +1SD / +2SD / p95 extras exist but default OFF (toggles below).
//  • Causal: a Monday's own move NEVER feeds its own lines — live stats are
//    frozen at the base latch using only previously completed Mondays.
//  • Percent mode only (fixed-$ levels don't scale across BTC's price range).
//  • Live sets need chart history: ~5 weekends on screen for the 4-Monday
//    set, ~14 for the 3-month set → use 15m / 30m / 1H charts (1m won't
//    hold enough bars; the live lines just stay empty until warmed up).
//  • FIX 2026-07-13: Monday base is now latched at EXACTLY the dead-gap end
//    (02:00 Dubai). The old straddling-bar rule grabbed close[1] one bar
//    early, so a 1h chart anchored 60 min before the boundary and a 15m
//    chart 15 min before it → every Monday line printed at different prices
//    per timeframe (observed gap up to ~$1,300). All 30 level lines and the
//    live 4-Monday / 3-month stats share this base, so all are fixed at once.
// ============================================================================

// =====================
// Inputs
// =====================
weeks_to_keep      = input.int(4, "How many weekends to keep", minval=1, maxval=12)
show_weekend_lines = input.bool(true, "Show weekend high/low lines")

// Timezones + weekend window (unchanged from the original)
tz_start   = input.string("America/New_York", "Weekend START timezone (NY)")
tz_end     = input.string("Asia/Dubai",       "Weekend END timezone (Dubai)")
start_hour = input.int(17, "Weekend START hour (NY session close)", minval=0, maxval=23)
start_min  = input.int(0,  "Weekend START minute",                  minval=0, maxval=59)
end_hour   = input.int(2,  "Weekend END hour (Dead Gap END, Dubai)", minval=0, maxval=23)
end_min    = input.int(0,  "Weekend END minute",                     minval=0, maxval=59)

// Level-set toggles
show_hist = input.bool(true, "Show ALL-HISTORY lines",          group="Level sets")
show_4wk  = input.bool(true, "Show LAST-4-MONDAYS lines (live)", group="Level sets")
show_3mo  = input.bool(true, "Show LAST-3-MONTHS lines (live)",  group="Level sets")

// Optional extras — apply to every visible set, default OFF
show_sd1 = input.bool(false, "Show +1SD lines", group="Extras (default off)")
show_sd2 = input.bool(false, "Show +2SD lines", group="Extras (default off)")
show_p95 = input.bool(false, "Show p95 lines",  group="Extras (default off)")

// ALL-HISTORY constants (%) — Truth Report 2026-07-12, 277 weekends 2019-2024
hist_up_mean = input.float(2.8076, "All-history UP mean %",   group="All-history constants")
hist_up_sd   = input.float(3.5904, "All-history UP SD %",     group="All-history constants")
hist_up_p95  = input.float(8.5684, "All-history UP p95 %",    group="All-history constants")
hist_dn_mean = input.float(2.7809, "All-history DOWN mean %", group="All-history constants")
hist_dn_sd   = input.float(3.2868, "All-history DOWN SD %",   group="All-history constants")
hist_dn_p95  = input.float(7.5895, "All-history DOWN p95 %",  group="All-history constants")

// =====================
// Helpers
// =====================
dayMs = 24 * 60 * 60 * 1000

// TRUE on the first bar that OPENS at/after ts (previous bar opened before).
// On that bar, close[1] is the close of the bar that ended exactly AT ts —
// the same price on every timeframe that aligns to the hour (1m…1h).
f_boundary(int ts) =>
    not na(time[1]) and time[1] < ts and time >= ts

// mean / sample-SD (ddof=1) / p95 of the LAST n values of src; na until n exist
f_last_stats(array<float> src, int n) =>
    int sz = array.size(src)
    float m = na
    float s = na
    float p = na
    if sz >= n
        array<float> tmp = array.new_float()
        for j = sz - n to sz - 1
            array.push(tmp, array.get(src, j))
        m := array.avg(tmp)
        float acc = 0.0
        for j = 0 to n - 1
            float d = array.get(tmp, j) - m
            acc += d * d
        s := n > 1 ? math.sqrt(acc / (n - 1)) : na
        p := array.percentile_linear_interpolation(tmp, 95)
    [m, s, p]

// =====================
// Rolling store of completed Monday moves (max 13 = the 3-month window)
// =====================
var array<float> up_moves = array.new_float()
var array<float> dn_moves = array.new_float()

// Live stats, recomputed every bar from PRIOR completed Mondays only
[u4m,  u4s,  u4p]  = f_last_stats(up_moves, 4)
[u13m, u13s, u13p] = f_last_stats(up_moves, 13)
[d4m,  d4s,  d4p]  = f_last_stats(dn_moves, 4)
[d13m, d13s, d13p] = f_last_stats(dn_moves, 13)

// =====================
// Storage for last N completed weekends (+ their frozen live-stat snapshots)
// =====================
var array<int>   wk_start_ts_arr = array.new_int()
var array<int>   wk_end_ts_arr   = array.new_int()
var array<int>   mon_end_ts_arr  = array.new_int()
var array<float> wk_high_arr     = array.new_float()
var array<float> wk_low_arr      = array.new_float()
var array<float> base_close_arr  = array.new_float()
// snapshots frozen at each weekend's base latch (causal)
var array<float> s_u4m  = array.new_float()
var array<float> s_u4s  = array.new_float()
var array<float> s_u4p  = array.new_float()
var array<float> s_u13m = array.new_float()
var array<float> s_u13s = array.new_float()
var array<float> s_u13p = array.new_float()
var array<float> s_d4m  = array.new_float()
var array<float> s_d4s  = array.new_float()
var array<float> s_d4p  = array.new_float()
var array<float> s_d13m = array.new_float()
var array<float> s_d13s = array.new_float()
var array<float> s_d13p = array.new_float()

f_trim_hist(int keep) =>
    while array.size(wk_start_ts_arr) > keep
        array.shift(wk_start_ts_arr)
        array.shift(wk_end_ts_arr)
        array.shift(mon_end_ts_arr)
        array.shift(wk_high_arr)
        array.shift(wk_low_arr)
        array.shift(base_close_arr)
        array.shift(s_u4m)
        array.shift(s_u4s)
        array.shift(s_u4p)
        array.shift(s_u13m)
        array.shift(s_u13s)
        array.shift(s_u13p)
        array.shift(s_d4m)
        array.shift(s_d4s)
        array.shift(s_d4p)
        array.shift(s_d13m)
        array.shift(s_d13s)
        array.shift(s_d13p)

f_trim_hist(weeks_to_keep)

// =====================
// Current (forming) weekend state
// =====================
var bool  weekend_active = false
var float cur_wk_high    = na
var float cur_wk_low     = na
var int   cur_wk_start   = na
var bool  base_latched   = false
var float cur_base_close = na
var int   cur_mon_end_ts = na

// Monday-window tracking (feeds the live stats when the Monday completes)
var bool  mon_active  = false
var float mon_hi      = na
var float mon_lo      = na
var float mon_base    = na
var int   mon_end_ts  = na

// =====================
// Detect weekend START (Friday NY close)
// =====================
int nyY = year(time, tz_start)
int nyM = month(time, tz_start)
int nyD = dayofmonth(time, tz_start)

int startTS_today = timestamp(tz_start, nyY, nyM, nyD, start_hour, start_min)
bool isFridayStartMoment = (dayofweek(startTS_today, tz_start) == dayofweek.friday)
bool startCross = isFridayStartMoment and f_boundary(startTS_today)

// =====================
// Detect weekend END (Dead Gap END on Monday, Dubai)
// =====================
int duY = year(time, tz_end)
int duM = month(time, tz_end)
int duD = dayofmonth(time, tz_end)

int deadGapEndTS_today = timestamp(tz_end, duY, duM, duD, end_hour, end_min)
bool isMondayDeadGapMoment = (dayofweek(deadGapEndTS_today, tz_end) == dayofweek.monday)
bool deadGapEndCross = isMondayDeadGapMoment and f_boundary(deadGapEndTS_today)

// =====================
// FIRST: finish a tracked Monday (push its move BEFORE any new latch below,
// so stats never contain a Monday that hasn't fully closed)
// =====================
if mon_active
    if time >= mon_end_ts
        if not na(mon_base) and mon_base > 0
            array.push(up_moves, (mon_hi - mon_base) / mon_base * 100.0)
            array.push(dn_moves, (mon_base - mon_lo) / mon_base * 100.0)
            while array.size(up_moves) > 13
                array.shift(up_moves)
            while array.size(dn_moves) > 13
                array.shift(dn_moves)
        mon_active := false
    else
        mon_hi := math.max(mon_hi, high)
        mon_lo := math.min(mon_lo, low)

// =====================
// State transitions
// =====================
if startCross
    weekend_active := true
    cur_wk_start := startTS_today
    cur_wk_high := high
    cur_wk_low  := low
    base_latched := false
    cur_base_close := na
    cur_mon_end_ts := na

// The end-firing bar opens AT/AFTER the dead-gap end — it is a Monday bar,
// so its high/low must not feed the weekend range.
if weekend_active and not deadGapEndCross
    cur_wk_high := na(cur_wk_high) ? high : math.max(cur_wk_high, high)
    cur_wk_low  := na(cur_wk_low)  ? low  : math.min(cur_wk_low,  low)

if weekend_active and deadGapEndCross
    if not base_latched
        // close[1] = close of the bar that ended exactly at 02:00 Dubai —
        // identical on every aligned timeframe (this is THE base for all lines)
        cur_base_close := close[1]
        base_latched := true

    int monMid = timestamp(tz_end, duY, duM, duD, 0, 0)
    cur_mon_end_ts := monMid + dayMs

    // store completed weekend + FREEZE the live stats as they are right now
    // (they contain only prior completed Mondays — causal by construction)
    array.push(wk_start_ts_arr, cur_wk_start)
    array.push(wk_end_ts_arr,   deadGapEndTS_today)
    array.push(mon_end_ts_arr,  cur_mon_end_ts)
    array.push(wk_high_arr,     cur_wk_high)
    array.push(wk_low_arr,      cur_wk_low)
    array.push(base_close_arr,  cur_base_close)
    array.push(s_u4m, u4m)
    array.push(s_u4s, u4s)
    array.push(s_u4p, u4p)
    array.push(s_u13m, u13m)
    array.push(s_u13s, u13s)
    array.push(s_u13p, u13p)
    array.push(s_d4m, d4m)
    array.push(s_d4s, d4s)
    array.push(s_d4p, d4p)
    array.push(s_d13m, d13m)
    array.push(s_d13s, d13s)
    array.push(s_d13p, d13p)

    f_trim_hist(weeks_to_keep)

    // start tracking THIS Monday so its move joins the stats when it closes
    mon_active := true
    mon_base   := cur_base_close
    mon_hi     := high
    mon_lo     := low
    mon_end_ts := cur_mon_end_ts

    // reset weekend state
    weekend_active := false
    cur_wk_high := na
    cur_wk_low  := na
    cur_wk_start := na
    base_latched := false
    cur_base_close := na
    cur_mon_end_ts := na

// Build plot series
// =====================

float wk_lo_series = na

float h_up_mean = na
float h_up_sd1 = na
float h_up_sd2 = na

    wk_lo_series := cur_wk_low

int sz = array.size(wk_start_ts_arr)
int i = 0
while i < sz

    int   e_ts  = array.get(wk_end_ts_arr, i)
